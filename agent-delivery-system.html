01 / Output

卜灵是第一个真实产出。

卜灵不是交付底座本身，而是这套底座服务的产品结果。项目 README 记录的当前事实是：前端 Flutter + Provider，后端 LangGraph + FastAPI，数据库 PostgreSQL，缓存/队列 Redis + Celery，支持六爻、梅花易数、八字、水占、筊杯、合盘和多位 AI 大师风格解读。

Product stack
Flutter 3.x + Dart + Provider；LangGraph + FastAPI + Uvicorn；PostgreSQL 15；Redis 7 + Celery；JWT；Firebase Cloud Messaging。

API surface
/divination/{type}/interpret、/masters、/sessions、/auth，明确禁止额外 /api 或版本号前缀。

Demo ready

03 / Workflow

MuYu/CLAUDE.md 是交付协议，不是备忘录。

MuYu 项目把 Claude 定位成 CTO、PM 和最大用户，Codex 承担高级工程师执行位。CLAUDE.md 的核心不是“提示词写得好”，而是强制开发进入一条能验证、能部署、能发版的工作流。

开始 = 写计划用户说“开始”后，Claude 用 superpowers:writing-plans 输出结构化计划到 tasks/todo.md，不是直接写代码。
前端先过设计规范任何新页面、UI 改动、交互设计都必须在计划后触发 /ui-ux-pro-max；纯后端任务才跳过。
Codex 审计划/codex-plan-review 最多 5 轮，重点看验收标准、影响面、存量兼容和闭环验证节点。
Codex 执行代码/codex-exec 逐 task 实现；Claude 不直接写代码，负责规划、审查和拍板。
Claude 静态审查/claude-static-review 做编译、规范和设计一致性检查，前端改动不能绕过 UI 审查。
Codex 功能测试/codex-func-test 用 XcodeBuildMCP、curl、日志和模拟器实际路径证明功能跑通。
远程部署和发版/deploy-remote 合入 main、部署生产服务器并线上测试；用户确认后 /testflight-deploy 上传 TestFlight。

Red line
flutter analyze 通过不等于测试通过；没有闭环验证，不能标记完成，不能部署。

Branch discipline
非 trivial 改动走 feature branch；每个 task 一个 commit；推分支、远程部署验证后再合 main。

04 / Team

muyu-claw-team 把 Agent 变成团队。

这个仓库的核心原则很克制：一个 bot 映射一个 agent，不做自动路由；私聊、群聊和定时任务会话隔离；60 秒内的新消息只合并进尚未开始的 pending task。它解决的是“多人、多 bot、多任务如何稳定进入执行队列”的问题。

Runtime flow
bridge event → normalize → resolve bot/agent/session → enqueue → immediate ack → worker → workspace projection → sandbox → Claude/Codex runtime → final delivery。

State boundary
Redis 和 PostgreSQL 承接队列、会话、群组记录和持久化状态；runtime 权限边界交给选定沙箱。

05 / Infra A

cui-no-auth 解决“看不见任务”的问题。

Claude Code 长任务不应该只能在终端里盯着。cui-no-auth 通过浏览器展示会话列表、详情更新和 sub-agent 执行过程，团队成员可以一起看同一条任务进度。

Directory Discovery每 15 秒扫描 ~/.claude/projects/**，发现新的 *.jsonl 并建立 sessionId 到文件路径的索引。
File Polling对已知 session 比较 mtime 和 size，推断首页列表和详情页是否需要更新。
SSE Broadcaster通过 session_list_update 和 session_content_update 把变化推到浏览器。
06 / Infra B

model-steer-claude 解决“模型断档”的问题。

长任务真正怕的不是慢，而是 429、500、额度耗尽之后整条链路断掉。NB-Claude 架在 Claude Code 和模型 API 之间，把 fallback、错误分类、熔断探测和任务路由放进同一个本地代理。

Never Fallback
主模型不可用时自动降级到下一个 provider；错误分类区分临时限流、服务异常、内容安全和真实配额耗尽。

Task Routing
cr --route 把路由规则注入会话，简单任务、编码任务、架构任务可以切到不同模型等级。

07 / Repos

三个仓库对应三个 Forge 能力。

它们不是 04 Outputs 的作品卡，而是 03 Forge 的工程底座：团队执行、任务可观测、模型续航。

Team

muyu-claw-team

Personal multi-bot agent system，内置 dev、ops、design 角色，支持私聊、群聊和定时任务。

 Observability

cui-no-auth

Claude Code Web 实时监控平台，自动发现会话、刷新详情并支持 sub-agent 钻取。

model-steer-claude

Claude Code 高可用模型代理，提供 fallback、circuit breaker、额度查看和任务路由。

判断标准很简单：能不能被交付、被观察、被恢复。

如果 AI 编程只是一次聊天，它很难稳定生产。这里的方向是把它变成一个有队列、有角色、有日志、有模型续航、有发布验收的工程系统。
